# My Hive

### **MY HIVE – Facilitator Dashboard**

**Purpose:**  
 A sacred workspace inside FloreSer.Life where each Pollinator can manage their offerings, sessions, and growth within the ecosystem.

---

## **1\. Application & Onboarding Flow**

**Why it matters:** Builds trust and consistency across the ecosystem. Ensures quality and alignment.

**Should include:**

* **Application Form:** Custom intake with archetype selection, bio, modality list, and sample session video (to assess presence, not just credentials).

* **Vetting Process:**

  * Step 1: Submission review (admin or peer mentors).

  * Step 2: Optional live mini-interview.

  * Step 3: Decision \+ archetype/level assignment (Rising, Evolving, Wise).

* **Agreements:** Digital signature for Terms of Use, Ethical Code, and Communication Guidelines (aligned with the Code of Conduct from the Sitemap).

* **Welcome Email:**

  * Warm poetic tone reflecting FloreSer voice.

  * Include access links \+ orientation video.

* **Onboarding Kit:**

  * **Videos:** “Welcome to FloreSer” \+ “Mentoring Circle: How to Thrive as a Pollinator.”

  * **Templates:** Session outlines, client follow-up message samples, community post guide.

  * **Branding Kit:** Logos, font references, color palette, framing rules.

  * **Co-Advertising Templates:** Reusable Canva links (Reels \+ carousel frames).

  * **Facilitator FAQ PDF:** Platform use, payout, and ethics.

**Suggestion:** Add an **“Orientation Call” option** for new facilitators (monthly with a mentor Pollinator). This deepens belonging and quality alignment.

---

## **2\. Soul Profile (Public View)**

**Why it matters:** This is what clients see. It needs emotional resonance *and* clarity.

**Should include:**

* **Archetype Badge \+ Level Color** (Bee, Butterfly, Beetle, Hummingbird).

* **Short Story / Bio:** Poetic but concrete—what brought them here.

* **Offerings:** Clear titles \+ 1-line poetic description \+ duration \+ price.

* **Testimonials:** Curated quotes from verified clients.

* **Languages Spoken** and **Location (optional).**

* **Media:** Profile photo \+ optional banner or video introduction.

**Suggestion:** Include a “*Facilitator Essence Quote*” — a single line displayed on hover, reinforcing brand voice (e.g., *“I help you listen to the quiet within.”*).

---

## **3\. Session Management Tools**

**Why it matters:** Smooth logistics \= credibility and peace of mind.

**Features:**

* Calendar integration (Google / native).

* Manage availability, rescheduling, cancellations, and session status.

* Automated reminders to both parties.

* Option for in-app or Zoom session link generation.

* Post-session summary field (private notes).

* Option to mark “Follow-up Recommended” → triggers client message template.

**Suggestion:** Add **“Energy Level Tracker”** — facilitator marks their availability by energy level (High / Balanced / Resting). This keeps authenticity and prevents burnout.

---

## **4\. Payments & Invoicing**

**Why it matters:** Transparency builds trust and professionalism.

* Split-payment integration via Checkout.com (as confirmed in Sitemap).

* Dashboard overview:

  * Total earnings

  * Upcoming payouts

  * Pending invoices

  * Commission % by facilitator tier (Founder / Rising / Core).

* Auto-generated invoices (client & facilitator copies).

* Visual summary: monthly revenue graph.

**Suggestion:** Add **“Session Value Reflection”** — optional field for facilitator to record personal notes (“What worked / what could improve”). It helps mentoring and growth tracking later.

---

## **5\. Messaging & Relationship Tools**

**Why it matters:** Builds trust, safety, and continuity.

* In-app chat with message templates:

  * “Thank you for your session.”

  * “Follow-up invitation.”

  * “Check-in after 7 days.”

* Pre-session client form access (from My Garden).

* File sharing (PDFs, audio meditations).

* Read-only transcript or recording notes (optional client view).

**Suggestion:** Include a **“Reflections Journal”** space—facilitator can leave insights to revisit patterns across clients (private, never shared).

---

## **6\. Collaboration & Referral Tools**

* **Refer Another Facilitator:** Generate referral link or in-platform nomination form.

* **Cross-Pollinate:** Suggest joint sessions or co-facilitations (e.g., Sound \+ Movement).

* **Mentor Path:** Option to mentor Rising Pollinators (badge \+ reward).

* **Upload to Community Garden:** Contribute free or member-only content (videos, articles, rituals).

**Suggestion:** Add **“Visibility Rewards”** linked to contribution:

* Feature in “Facilitator of the Month” carousel.

* Boosted profile visibility in search results.

* Reduced platform fee.

---

## **7\. Analytics & Growth Tracking**

**Why it matters:** Facilitators stay motivated when they see their impact.

* Dashboard overview: sessions completed, retention rate, average rating.

* Insight cards: “Most booked day,” “Returning clients,” “Most resonant offering.”

* Optional “Flourishing Index”: gamified metric reflecting contribution to The Garden, testimonials, and client satisfaction.

**Suggestion:** Include **badge progression** (Rising → Evolving → Wise) tied to time, sessions, and client feedback, reinforcing the journey concept.

---

## **8\. Community Integration**

* Access to Founder/Mentor Circle Calls (invite-only for top facilitators).

* Calendar of facilitator-only workshops or co-creation calls.

* News & Rituals section: Seasonal musings, upcoming campaigns, or co-ads reminders.

**Suggestion:** Add an **“Internal Notice Board”** — to share open calls, collaborations, or initiatives (like the Open Bloom Festival).

// Analytics flower (4 petals for key KPIs)  
function AnalyticsFlower({ values \= \[0.7,0.6,0.5,0.4\] }) {  
  const size \= 180; const cx \= size/2, cy \= size/2; const r \= size\*0.38; const petals \= values.length;  
  const petalsPath \= values.map((v,i)=\>{  
    const a \= (Math.PI\*2\*i)/petals \- Math.PI/2; const R \= r\*(0.55+v\*0.45);  
    const x \= cx \+ Math.cos(a)\*R; const y \= cy \+ Math.sin(a)\*R;  
    const c1x \= cx \+ Math.cos(a-0.45)\*R\*0.6; const c1y \= cy \+ Math.sin(a-0.45)\*R\*0.6;  
    const c2x \= cx \+ Math.cos(a+0.45)\*R\*0.6; const c2y \= cy \+ Math.sin(a+0.45)\*R\*0.6;  
    return {x,y,c1x,c1y,c2x,c2y};  
  });  
  return (  
    \<svg width={size} height={size} viewBox={\`0 0 ${size} ${size}\`} className="drop-shadow-sm"\>  
      \<circle cx={cx} cy={cy} r={r+12} className="fill-\[\#FFE6B0\]"/\>  
      {petalsPath.map((p,i)=\> (  
        \<path key={i} d={\`M ${cx} ${cy} C ${p.c1x} ${p.c1y}, ${p.c2x} ${p.c2y}, ${p.x} ${p.y} Z\`} className="fill-amber-400/90"/\>  
      ))}  
      \<circle cx={cx} cy={cy} r={18} className="fill-white stroke-amber-500"/\>  
    \</svg\>  
  );  
}
